# Block

# Block是如何实现的？Block对应的数据结构是什么样子的？__block的作用是什么？

block本质是一个对象，底层用struct实现。

struct Block_layout {
    void *isa;// isa 指针
    int flags;// 用于按 bit 位表示一些 block 的附加信息
    int reserved;// 保留变量
    void (*invoke)(void *, ...);// 函数指针，指向具体的 block 实现的函数调用地址
    struct Block_descriptor *descriptor;//  表示该block的附加描述信息,主要是size大小，以及copy和dispose函数的指针
    /* Imported variables. */ // 捕获过来的变量 
};

__block的作用是可以获取对应变量的指针，使其可以在block内部被修改。

# 为什么当我们在使用block时外面是weak 声明一个weakSelf，还要在block内部使用strong再持有一下？

block外界声明weak是为了实现block对对象的弱持有，而里面的作用是为了保证在进到block时不会发生释放

# 三种block

  • NSGlobalBlock 静态区block,这是一种特殊的block,因为不引用外部变量而存在。另外,作为静态区的对象,它的释放是有操作系统控制的。
  • NSStackBlock 栈区block,位于内存的栈区,一般作为函数的参数出现。
  • NSMallocBlock 堆区block,位于内存的堆区,一般作为对象的property出现。

# OS中block 捕获外部局部变量实际上发生了什么？__block 中又做了什么？

block 捕获的是当前在block内部执行的 外部局部变量的值，并没有捕获到它的内存地址

说白了， block内部可以修改的是堆中的内容， 但不能直接修改栈中的任何东西。

如果加上__block 在运行时创建了一个外部变量的“副本”属性，把栈中的内存地址放到了堆中进而在block内部也能修改外部变量的值。
